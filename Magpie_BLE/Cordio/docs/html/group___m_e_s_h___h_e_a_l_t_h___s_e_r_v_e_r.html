◆ MeshHtSrSignalTestEnd()

void MeshHtSrSignalTestEnd ( meshElementId_t  elementId,
uint16_t  companyId,
meshAddress_t  meshHtClAddr,
uint16_t  appKeyIndex,
bool_t  useTtlZero,
bool_t  unicastReq 
)

Signals to a Health Client that a test has been performed.

Parameters
[in]elementIdHealth Server element identifier to determine instance.
[in]companyIdCompany identifier for determining fault state.
[in]meshHtClAddrHealth Client address received in the MESH_HT_SR_TEST_START_EVENT.
[in]appKeyIndexAppKey identifier received in the MESH_HT_SR_TEST_START_EVENT.
[in]useTtlZeroTTL flag received in the MESH_HT_SR_TEST_START_EVENT.
[in]unicastReqUnicast flag received in the MESH_HT_SR_TEST_START_EVENT.
Returns
None.
Remarks
Upon receiving a MESH_HT_SR_TEST_START_EVENT event with the notifTestEnd parameter set to TRUE, the user shall call this API using the associated event parameters, but not before logging 0 or more faults so that the most recent test identifier is stored.
